## Parser-Critical Findings

### 1. MEMO FIELD PATTERNS

There are **4 distinct memo patterns**:

| Pattern | Example | Meaning | Token Types |
|---------|---------|---------|-------------|
| `DOVU:<SYMBOL>:<topic_id>` | `DOVU:SUMMERLEY:0.0.617740` | Carbon credit token linked to HCS topic | Original farm tokens |
| `<topic_id>` (bare) | `0.0.1687273` | Topic reference only (no prefix) | Green Bond, OTC, ELV Certificate |
| Free text | `"dNFT 6 dp guardian"`, `"#bloodmoon"` | Descriptive/experimental | Test/experimental tokens |
| Empty `""` | | No memo set | ALL Guardian reissue tokens, DOVU utility tokens |

**Pattern regex for DOVU credits:** `^DOVU:([A-Z0-9-]+):(0\.0\.\d+)$`
**Pattern regex for bare topic:** `^(0\.0\.\d+)$`

### 2. NAME PATTERNS

| Pattern | Example | Token Category |
|---------|---------|---------------|
| `"<FarmName>"` | `"Summerley Hall Fruit Farm"` | Original carbon credit |
| `"<FarmName> (guardian-v1-reissue)"` | `"Briyastovo Wheat Farm (guardian-v1-reissue)"` | Guardian reissue v1 |
| `"<FarmName> (Guardian Reissue)"` | `"Redhill (Guardian Reissue)"` | Guardian reissue v2+ |

### 6. NFT METADATA FORMAT

**Guardian Reissue NFTs (0.0.1658305, 0.0.1719752):**
- Metadata is base64-encoded **Hedera timestamps** (e.g., `1672855650.725381885`)
- NOT IPFS CIDs
- All serials within a batch share the same timestamp
- Timestamp likely represents the minting transaction time

**Green Bond Certificate NFT (0.0.1687320):**
- Metadata IS an IPFS CID: `ipfs://bafkreiavbc3aivdlds4bhzxpwm2hu4jkfcpwv6ndbfcm2zhzu43lpovdmu`
- IPFS content unreachable via public gateways (403/504 errors)
- `bafkrei` prefix = CIDv1 with raw codec (single file, not DAG)

**ELV Certificate NFTs (0.0.4314637):**
- Metadata is base64-encoded **Hedera timestamps**
- Same pattern as Guardian reissue NFTs

### 7. TIMESTAMPS

- `created_timestamp` — token creation (Hedera consensus timestamp, seconds.nanoseconds)
- `modified_timestamp` — last modification (supply change, key update, etc.)
- `expiry_timestamp` — token expiry (nanosecond precision, usually ~90 days after creation)
- NFT-level `created_timestamp` and `modified_timestamp` per serial
- No geography/location data anywhere in memo, metadata, or token-level fields
